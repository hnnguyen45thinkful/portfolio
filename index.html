Here’s what I build. MY PROJECTS

Lookitup

· Contextualized word search · Open API use
· AJAX calls · jQuery traversal & DOM manipulation

Lookitup is a word/phrase-search app that goes above and beyond a dictionary definition and saves you the hassle of opening up new web pages to view search results. A search with Lookitup grabs: relevant dictionary entries with any pronunciation audio files and usage examples available in Longman’s dictionary; the introduction section of the search terms’s Wikipedia page; and finally YouTube search results, specifically tweaked to filter for descriptive and informative clips and instantly playable on an on-page pop-up player without having to leave the search results.

Collabowrite

· Real-time events · React · ES6· Node.js
· Express · MongoDB/Mongoose · Webpack

Collabowrite is an app that enables users to work together real-time on sentence/paragraph-level writing. It uses account-free sign-up and sign-in by generating a random, unique project access code when a user first creates a project. The code can then be shared to let others join and contribute to the project by providing feedback, revisions, or new proposals, which get displayed real-time to all users in session. Combined with a mobile-friendly interface, Collabowrite is great for inviting participation from a large audience during presentations and class.

STU-DEX

· Roster management system · Node.js
· Express· MongoDB/Mongoose · Mocha · Chai

STU-DEX is a roster management system for teachers with a diverse student demographic (e.g. ESL or international college/university programs). Teachers can have students fill out surveys about where they’re from, what they’re studying or pursuing as a career, and other interesting facts they want to share. STU-DEX then generates printable weekly rosters that show student-submitted profiles to help teachers quickly learn their students’ names and who they are. The app also enables teachers to easily communciate with students by e-mail, individually or class-wide.

Here’s how I build. MY SKILLS


Front-End Development

· HTML5 · CSS3 · SASS · Bootstrap
· JS · jQuery · ES6 · React · Redux

I can build and style an interactive, responsive webpage or app interface from scratch (like this one) with HTML, CSS, and jQuery, or using a CSS framework like Bootstrap. For full-stack apps, I’m a fan of writing modular JS components in ES6 using React as a front-end framework and Redux to manage state.


Back-End Development

· RESTful APIs · Node.js · Express
· MongoDB · Mongoose

I can build full-stack apps with RESTful APIs that perform CRUD operations using Node.js and Express as my server-side programming framework. I use JSON data in my applications and MongoDB as my database along with Mongoose to create data schema.


Real-time & Authentication

· Pusher · Socket.IO
· Passport.js · bcrypt

I can build apps that have real-time event triggering to notify users of new events or actions. I’ve used Pusher to build an app that alerts and updates user screens real-time with new posts and comments, and Socket.io to build a live chatroom. For user log-in, I use Passport.js as an authentication framework and bcrypt for password hashing.



Development Environment

· Cloud9 IDE · Chrome DevTools
· command-line · Git · npm · Webpack
· Mocha · Chai · Travis CI · Heroku

I work on Cloud9, a cloud-based IDE that allows me to work from anywhere, and I use Chrome DevTools for trouble-shooting errors and analyzing front-end elements. I use npm to install and manage dependencies that are utilized in my apps, and Webpack to generate a bundle JS file that consist of all modules with dependencies. Finally, I use Mocha, Chai, and Travis for testing and deploy my apps on Heroku.


Architecture Principles

· Responsive web design
· Mobile-first development
· Cross-browser compatibility

All of my work takes into consideration that users view from different browsers and devices that vary in screen size. I build webpages and apps to be stylistically responsive to the screen size by using CSS media queries and functionally responsive to screen size as well as browser type using JS to identify viewport size, OS and browser type. By developing first for mobile devices, I ensure that functionality is not lost in any sized device, and lastly I test the user experience in different browsers for cross-browser compatibility.

Here’s why I build. ABOUT ME

#EdTech Most of us live and breathe technology today, and we certainly do not stop doing so when we step into a classroom or sit down to learn something new. So as an educator at the university level, I’ve always tried to integrate intuitive technology into instruction to create more opportunities for proactive and interactive learning. With a solid understanding of the overall process of full-stack development, gained by building quality projects myself, I now hope to make use of my uniquely qualified perspectives and skillsets to be a part of the development next-generation EdTech.

#ProblemSolving I enjoy a good challenge, and the fact that software development = problem solving—from identifying the process your product aims to streamline, to trouble-shooting errors in the code and functionality—instantly drew me in as I started building my own full-stack projects. It can feel like a puzzle sometimes: coordinating communication between front-end and back-end components, all the while considering details in the presentation and not losing sight of the big picture purpose of the product. But good news: puzzles are fun.

#UI+UXDisclaimer: I’m obsessed with details (in a good way). Recalling my own experience as a user, I really think that it often comes down to the UI and design elements that makes users give something a serious try. As such, I’m a firm believer that clean and beautiful presentation is just as important as easy-to-use functionality, so fine-tuning how the product shows up and interacts with the user is always something I spend extra time on. The importance placed on UI/UX should, of course, multiply when the software is meant to facilitate real learning, and that’s where the teacher-me kicks in again.

#CreativityBut at the end of the day, I’m really just excited about the creative process of building. Time flies when I’m envisioning the look and feel of the application, how the user should be able to navigate it, and the most effective and efficient approaches to code its functionality. I believe that creativity thrives and quality evolves when a passionate and invested group of people work together, and I’m ready to play my part in a team where my experiences to date can be a genuine contribution.
