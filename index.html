Here’s what I build. MY PROJECTS

Lookitup.co

· Contextualized word search · Open API use
· AJAX calls · jQuery traversal & DOM manipulation

Lookitup.co is a word/phrase-search app that goes above and beyond a dictionary definition and saves you the hassle of opening up new web pages to view search results. A search with Lookitup.co grabs: relevant dictionary entries with any pronunciation audio files and usage examples available in Longman’s dictionary; the introduction section of the search terms’s Wikipedia page; and finally YouTube search results, specifically tweaked to filter for descriptive and informative clips and instantly playable on an on-page pop-up player without having to leave the search results.

Collabowrite.co

· Real-time events · React · ES6· Node.js
· Express · MongoDB/Mongoose · Webpack

Collabowrite.co is an app that enables users to work together real-time on sentence/paragraph-level writing. It uses account-free sign-up and sign-in by generating a random, unique project access code when a user first creates a project. The code can then be shared to let others join and contribute to the project by providing feedback, revisions, or new proposals, which get displayed real-time to all users in session. Combined with a mobile-friendly interface, Collabowrite.co is great for inviting participation from a large audience during presentations and class.

STU-DEX.co

· Roster management system · Node.js
· Express· MongoDB/Mongoose · Mocha · Chai

STU-DEX.co is a roster management system for diverse classrooms (e.g. ESL, adult/continuing ed, higher ed). Students are able to add themselves to course rosters by completing an online survey about where they’re from, what they’re studying or pursuing as a career, and other interesting facts they want to share. Teachers can then generate weekly rosters that include student profiles to help them engage and learn more about their students. The platform also allows for teachers to easily communicate with students, individually or class-wide, using e-mail addresses collected in the online surveys.

Here’s how I build. MY DEV SKILLS


Front-End Development

· HTML5 · CSS3 · SASS · Bootstrap
· JS · jQuery · ES6 · React · Redux

I can build and style an interactive, responsive webpage or app interface from scratch (like this one) with HTML, CSS, and jQuery, or using a CSS framework like Bootstrap. For full-stack apps, I’m a fan of writing modular JS components in ES6 using React as a front-end framework and Redux to manage state.


Back-End Development

· RESTful APIs · Node.js · Express
· MongoDB · Mongoose

I can build full-stack apps with RESTful APIs that perform CRUD operations using Node.js and Express as my server-side programming framework. I use JSON data in my applications and MongoDB as my database along with Mongoose to create data schema.


Real-time & Authentication

· Pusher · Socket.IO
· Passport.js · bcrypt

I can build apps that have real-time event triggering to notify users of new events or actions. I’ve used Pusher to build an app that alerts and updates user screens real-time with new posts and comments, and Socket.io to build a live chatroom. For user log-in, I use Passport.js as an authentication framework and bcrypt for password hashing.



Development Environment

· Cloud9 IDE · Chrome DevTools
· command-line · Git · npm · Webpack
· Mocha · Chai · Travis CI · Heroku

I work on Cloud9, a cloud-based IDE that allows me to work from anywhere, and I use Chrome DevTools for trouble-shooting errors and analyzing front-end elements. I use npm to install and manage dependencies that are utilized in my apps, and Webpack to generate a bundle JS file that consist of all modules with dependencies. Finally, I use Mocha, Chai, and Travis for testing and deploy my apps on Heroku.


Architecture Principles

· Responsive web design
· Mobile-first development
· Cross-browser compatibility

All of my work takes into consideration that users view content from different browsers and devices that vary in screen size. I build webpages and apps to be stylistically responsive to the screen size by using CSS media queries and functionally responsive to screen size/browser type using JS to identify viewport size, OS and browser. By developing first for mobile devices, I ensure that functionality is not lost in any sized device, and lastly I test the user experience in different browsers for cross-browser compatibility.

Here’s why I build. ABOUT ME

#EdTech/Productivity Most of us live and breathe technology today, and we certainly don’t stop doing so when we step into a classroom or sit down to work. So as an educator at the university level, I’ve always integrated technology into the coursework — both in and out of class — to create more intuitive and engaging opportunities for students to learn and work together. My keen interest in tech solutions for education and productivity has evolved since then, making me transition from a user to a developer. Now with a solid understanding of the overall process of software development, gained by building quality products myself, I’m hoping to make use of my uniquely qualified perspectives and skills to participate in the conception and development of technology that improves the way we learn and work.

#ProblemSolving I enjoy a good challenge, and the fact that software development = problem solving—from identifying the process your product aims to streamline, to trouble-shooting errors in the code and functionality—instantly drew me in as I started building my own projects. It can feel like a puzzle sometimes: coordinating communication between front-end and back-end components, all the while considering UI/UX details and not losing sight of the big picture purpose of the product.
But good news: puzzles are fun.

#UserCenteredDevDisclaimer: I’m obsessed with details (in a good way). Recalling my own experience as a user, I really think that it often comes down to a clean, beautiful UI that makes users give something a serious try. Then comes the easy-to-use and simple user functionality. So putting myself in the user’s shoes during development, then fine-tuning the product with user feedback, is something I spend extra time on. The importance placed on UI/UX only multiplies when the users are students and the product is meant to facilitate learning or improve productivity — and that’s where my teacher instincts kick in.

#CreativityBut at the end of the day, I really just enjoy the creative process of envisioning solutions and developing products. Time flies when I start thinking of ways to streamline a process or solve a specific problem, drawing up user flows and wireframes, bouncing around ideas for specific functions, and ultimately building a simple but effective solution. I believe that creativity thrives and quality evolves when a passionate and invested group of people work together, and I’m ready to play my part in a team of enthusiasts where my experiences to date can be a genuine contribution.
