Here’s what I build. MY PROJECTS

Collabowrite

· Real-time events · React · ES2016· Node.js
· Express · MongoDB/Mongoose · Webpack

Collabowrite is an app that enables users to work together real-time on sentence/paragraph-level writing. It uses account-free sign-up and sign-in by generating a random, unique project access code when a user first creates a project. The code can then be shared to let others join and contribute to the project by providing feedback, revisions, or new proposals, which get displayed real-time to all users in session. Combined with a mobile-friendly interface, Collabowrite is great for inviting participation from a large audience during presentations and class.

Lookitup

· Contextualized word search · Open API use
· AJAX calls · jQuery traversal & DOM manipulation

Lookitup is a word/phrase-search app that goes above and beyond a dictionary definition and saves you the hassle of opening up new web pages to view search results. A search with Lookitup grabs: relevant dictionary entries with any pronunciation audio files and usage examples available in Longman’s dictionary; the introduction section of the search terms’s Wikipedia page; and finally YouTube search results, specifically tweaked to filter for descriptive and informative clips and instantly playable on an on-page pop-up player without having to leave the search results.

STU-DEX

· Course management system · Node.js
· Express· MongoDB/Mongoose · Mocha · Chai

STU-DEX is a course management system for teachers with a diverse student demographic (e.g. ESL or international college/university programs). Teachers fill out and save information about where their students are from, what they’re studying or pursuing as a career, and other interesting facts shared during icebreaking sessions. STU-DEX can then be used to send class-wide emails or print weekly rosters which include the saved student information along with space for attendance and notes to help teachers quickly learn their students’ names and backgrounds.

I can build full-stack apps with RESTful APIs that perform CRUD operations using Node.js and Express as my server-side programming framework. I use JSON data in my applications and MongoDB as my database along with Mongoose to create data schema.


Real-time & Authentication

· Pusher · Socket.io
· Passport.js · bcrypt

I can build apps that have real-time event triggering to notify users of new events or actions. I’ve used Pusher to build an app that alerts and updates user screens real-time with new posts and comments, and Socket.io to build a live chatroom. For user log-in, I use Passport.js as an authentication framework and bcrypt for password hashing.



Development Environment

· Cloud9 IDE · Chrome DevTools
· command-line · Git · npm · Webpack
· Mocha · Chai · Travis CI · Heroku

I work on Cloud9, a cloud-based IDE that allows me to work from anywhere, and I use Chrome DevTools for trouble-shooting errors and analyzing front-end elements. I use npm to install and manage dependencies that are utilized in my apps, and Webpack to generate a bundle JS file that consist of all modules with dependencies. Finally, I use Mocha, Chai, and Travis for testing and deploy my apps on Heroku.


Architecture Principles

· Responsive web design
· Mobile-first development
· Cross-browser compatibility

All of my work takes into consideration that users view from different browsers and devices that vary in screen size. I build webpages and apps to be stylistically responsive to the screen size by using CSS media queries and functionally responsive to screen size as well as browser type using JS to identify viewport size, OS and browser type. By developing first for mobile devices, I ensure that functionality is not lost in any sized device, and lastly I test the user experience in different browsers for cross-browser compatibility.
